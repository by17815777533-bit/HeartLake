# Changelog

HeartLake 项目变更记录，按类别组织。

## [0.9.0] - 2026-02-24

### 新功能

- **多视角亲密分算法** — 升级为多维度亲密分计算（互动频率 + 情感共鸣 + 时间衰减 + 互惠性），替代旧版单一计数模型
- **情绪热力图独立页面** — 从情绪日历中拆分热力图为独立的 `emotion_heatmap_screen.dart`
- **情绪趋势分析页面** — 新增 `emotion_trends_screen.dart`，展示情绪变化趋势
- **多专家情绪融合** — EdgeAI 情感分析升级为三层融合策略（规则 + 词典 + 统计），支持 ONNX 模型集成
- **两阶段向量检索** — HNSW 粗筛 + 精排，提升语义搜索精度
- **C++ 推理并发缓存** — EdgeAI 推理链路增加并发缓存层，提升吞吐量

### 修复

- **湖神情感链路修复** — 去除固定兜底话术，新增基于情绪与趋势的动态本地回退；湖神聊天同步写入情绪脉搏样本
- **情绪脉搏口径统一** — 公开接口补充 `normalized_score/trend`，并兼容 `score_scale=zero_one` 历史提交方式
- **ONNX 小模型兼容增强** — `EDGE_AI_ONNX_ENABLED` 支持显式开关与自动探测模式；`EDGE_AI_MODEL_PATH` 支持目录写法并自动解析模型与词表路径
- **跨域鉴权链路修复** — 认证前置与安全审计过滤器返回 `401/403` 时统一补齐 CORS 响应头
- **湖神回复重复修复** — `AIService::generateReply` 在 RAG 上下文禁用语义缓存复用
- **语义判定精度修复** — EdgeAI 增强失眠/焦虑词典，分词短语窗口扩展到 2~6 字
- **ONNX 路径健壮性修复** — 模型路径增加运行时相对路径回退
- **GPU 推理优先策略** — Ollama 调用默认注入 GPU 参数，ONNX 默认优先尝试 CUDA EP

### 下线

- **漂流瓶模式下线** — 彻底移除漂流瓶（drift bottle）相关功能，保留纸船（paper boat）作为唯一匿名通信方式

### 性能

- **WebSocket 离线队列优化** — 刷新性能从 O(n²) 优化到 O(n)
- **本地启动配置统一** — `start.sh` 升级为统一入口：自动生成运行时配置、自动拉起本地 PostgreSQL/Redis、自动迁移、自动检查/拉取 Qwen 模型

### 文档

- **老文档中文命名** — 统一评测报告输出路径
- **性能优化文档** — 补齐 EdgeAI 情绪推理并发缓存与评测链路文档

## [0.8.0] - 2026-02-21

### 新功能

- **ONNX Runtime 中文情感分析引擎** — 集成本地 ONNX 模型，支持离线中文情感分析 (`d15e05e`)
- **Ollama 原生 API 适配** — 支持 heartlake-qwen 自定义本地模型 (`b53f749`)
- **湖神聊天 API** — 添加湖神聊天端点，集成 EdgeAI 情感分析、内容审核和情绪脉搏 (`0ebea12`, `ec3eaba`)
- **匿名用户恢复关键词系统** — 256 词词库 + 6 词组合，使用 RAND_bytes (`0fc30e4`, `962da9a`)
- **心理咨询预约和会话页面** — 新增咨询师预约与会话 UI (`e958258`)
- **安全港湾独立页面** — 新增安全港湾功能页面 (`c11797b`)
- **纸船互动自动建立临时好友** — 纸船互动自动创建临时好友关系，移除手动添加按钮 (`ce461f5`)
- **好友聊天 WebSocket 实时消息** — 添加实时消息监听和乐观更新 (`2ad9f07`)
- **AI 可视化完善** — 集成相似石头推荐 + 双记忆 RAG 面板 (`891db84`)

- **角色命名统一** — 前端"点灯人"统一为"守护者"，用户可见"AI"文案统一为"湖神"
- **用户与纸船接口稳定性修复** — `users.username/recovery_key_hash` 兼容迁移补齐；`/api/users/my/boats` 返回结构兼容 `items + boats`
