## 三、后端 C++20 审计（78/100）

### 3.1 架构评价

**优点：**
- DDD 四层架构（Interfaces → Application → Domain → Infrastructure）分层清晰
- ServiceLocator 模式统一管理服务生命周期，`ArchitectureBootstrap` 启动时注册
- 事件驱动解耦业务模块（EventBus 发布-订阅）
- EdgeAI 8 大子系统算法完整度高，纯 C++ 实现无云端依赖
- PASETO v4 替代 JWT，消除算法混淆攻击面
- X25519 + AES-256-GCM 端到端加密

**问题：**

## 八、安全问题汇总（跨模块）

### 8.1 全栈 Token 安全链路

**当前状态：三端均存在 Token 泄露风险**

```
Flutter: SharedPreferences 明文存储 → URL query 传递 WebSocket Token
Admin:   localStorage 明文存储 → URL query 传递 WebSocket Token
Backend: PASETO v4 签发正确 → 但 AdminAuthFilter 放行 localhost Origin
```

**修复方案：**
1. Flutter: 迁移到 `flutter_secure_storage`（Android Keystore / iOS Keychain）
2. Admin: 改用 `httpOnly` cookie 存储 Token
3. 全端: WebSocket Token 改为连接后首条消息认证
4. Backend: 生产环境移除 localhost CORS 白名单

### 8.2 密码学安全

| 问题 | 风险 | 修复 |
|------|------|------|
| `RAND_bytes` 返回值未检查 | CSPRNG 失败时 IV 全零 | 检查返回值 |
| 恢复密钥无盐 SHA256 | 彩虹表攻击 | Argon2id 加盐 |
| 默认盐值 `CHANGE_ME` | 已知盐值 | 强制配置 |
| 48-bit 恢复密钥熵 | 暴力破解 | 增加到 8-12 词 |
