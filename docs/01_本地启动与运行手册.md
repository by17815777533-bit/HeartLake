# HeartLake 本地启动与运行手册

## 1. 目标
本手册用于在评审、演示、联调三种场景下，稳定启动 HeartLake 全链路环境。内容覆盖：
- 非 Docker 方式（便于开发调试、定位源码问题）
- Docker 方式（便于快速复现、交付验收）
- 启动顺序与每一步背后的技术原因

## 2. 运行架构与技术选型说明
- 后端：C++20 + Drogon，监听 `8080`。
  - 选择原因：高并发下延迟稳定，可直接控制线程与连接池，便于压测达标。
- 前端：Flutter Web（调试阶段使用 DDC）。
  - 选择原因：同一套业务逻辑可覆盖 Web/移动端，减少多端逻辑分叉。
- 数据库：PostgreSQL 16。
  - 选择原因：强事务一致性 + JSON 能力，适合社交内容与关系数据。
- 缓存：Redis 7。
  - 选择原因：高频读写（通知数、会话态、热数据）可降后端负载。
- 模型服务：Ollama（本地推理入口，默认 `11434`）。
  - 选择原因：本地可控、离线可跑、便于替换模型与压测稳定性。

## 3. 启动前检查
- 检查端口占用：
```bash
ss -ltnp | rg ':3000|:8080|:11434|:5432|:6379'
```
- 必需目录：
  - 后端源码：`backend/`
  - 前端源码：`frontend/`
  - 运行日志：`.runtime/logs/`

## 4. 非 Docker 启动（推荐开发态）

### 4.1 启动后端
```bash
cd /home/joker/code/heartlake/backend
./start.sh
```

技术说明：
- `start.sh` 会按迁移顺序补齐数据库结构，再构建并启动 `HeartLake`。
- 这样做的原因是保证“代码版本”和“数据库结构”一致，避免接口 500 由 schema 漏项触发。

健康检查：
```bash
curl -sS http://127.0.0.1:8080/api/health
```

### 4.2 启动前端
```bash
cd /home/joker/code/heartlake/frontend
flutter pub get
flutter run -d web-server --web-hostname 0.0.0.0 --web-port 3000
```

技术说明：
- `web-server` 方式便于局域网联调。
- 本项目在 `frontend/web/flutter_bootstrap.js` 中会在本地环境主动清理 Service Worker，避免旧缓存导致“页面不更新/白屏假象”。

访问地址：
- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:8080`

### 4.3 启动 Ollama（若未运行）
```bash
ollama serve
```

检查：
```bash
curl -sS http://127.0.0.1:11434/api/tags
```

## 5. Docker 启动（交付验收态）
```bash
cd /home/joker/code/heartlake
./scripts/start-services.sh all
```

技术说明：
- Docker 模式用于“快速起整套服务”，避免机器环境差异影响验收。
- 本地调试仍建议优先非 Docker，便于直接附加调试器和查看源码栈。

检查：
```bash
docker compose ps
curl -sS http://127.0.0.1:8080/api/health
```

## 6. 推荐启动顺序与原因
1. PostgreSQL / Redis
2. Ollama
3. 后端
4. 前端

原因：
- 后端启动时会初始化依赖与模型能力，若基础服务不可达会触发降级或启动失败。
- 前端最后启动可减少启动期无效请求与误报。

## 7. 常见问题与处理

### 7.1 前端白屏
排查顺序：
1. `3000` 是否在监听。
2. `8080` 是否在监听。
3. 浏览器控制台是否停在 `DDC is about to load ...`。
4. 强刷缓存 `Ctrl + Shift + R`。
5. 重启前端 `flutter run`。

### 7.2 接口 500
- 先看后端日志是否是 schema 缺失。
- 执行后端迁移后重启。

### 7.3 功能看起来“没生效”
- 优先确认 token 和 user_id 是否正确写入存储。
- 再确认请求是否命中旧缓存。

## 8. 关闭服务
- 前台进程：`Ctrl + C`
- 后台清理：
```bash
pkill -f '(^|/)HeartLake($| )' || true
pkill -f 'flutter_tools.snapshot run -d web-server' || true
pkill -f 'ollama serve' || true
```
