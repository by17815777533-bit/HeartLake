# 本地启动与运行手册

> 本文档提供 HeartLake 项目的两种部署方式：Docker 容器化部署（推荐）和本地开发环境部署。涵盖环境要求、启动流程、环境变量配置及常见问题排查。

## 1. 环境要求

| 组件 | 最低版本 | 用途 |
|------|----------|------|
| Linux | Ubuntu 22.04+ / Arch Linux | 开发与运行环境 |
| GCC / Clang | GCC 12+ / Clang 15+ | C++20 编译器 |
| CMake | 3.22+ | 构建系统 |
| PostgreSQL | 16 | 主数据存储 |
| Redis | 7 | 缓存服务 |
| Node.js | 18+ | 管理后台构建与运行 |
| Flutter | 3.19+（Dart 3.3+） | 移动客户端构建与运行 |
| Docker + Compose | Engine 20.10+ / Compose v2+ | 容器化部署 |
| Ollama | latest | 本地 LLM 推理（可选） |

## 2. Docker 容器化部署（推荐）

Docker Compose 编排 5-6 个容器，无需在宿主机安装任何开发依赖。

### 2.1 启动

```bash
cd HeartLake
cp .env.example .env    # 首次需要，后续可跳过
# 4GB+ 机器用 all；低配机器优先用 lite
./scripts/docker-up.sh lite
```

`docker-up.sh` 支持以下启动模式：

| 模式 | 命令 | 启动的服务 |
|------|------|-----------|
| `all` | `./scripts/docker-up.sh all` | 全部 6 个容器 |
| `lite` | `./scripts/docker-up.sh lite` | 除 Ollama 外的全部服务 |
| `server-lite` | `./scripts/docker-up.sh server-lite` | 2C2G Ubuntu 服务器精简模式 |
| `db` | `./scripts/docker-up.sh db` | 仅 PostgreSQL + Redis |

### 2.2 容器编排详情

| 服务 | 镜像 | 端口 | 健康检查 |
|------|------|------|----------|
| postgres | `postgres:16` | 5432 | `pg_isready -U postgres -d heartlake` |
| redis | `redis:7` | 6379 | `redis-cli -a <password> ping` |
| ollama | `ollama/ollama:latest` | 11434 | 无 |
| backend | 自构建（Ubuntu 24.04 多阶段） | 8080 | `curl -fsS http://127.0.0.1:8080/api/health` |
| admin | 自构建（Node 20 + Nginx 1.27） | 5173 | 无 |
| gateway | `nginx:1.27-alpine` | 3000 | 无 |

### 2.3 启动顺序

启动顺序通过 `depends_on` + `condition: service_healthy` 控制：

1. PostgreSQL 和 Redis 启动并通过健康检查
2. Backend 启动，自动执行数据库迁移
3. Admin 和 Gateway 启动

首次启动需构建 Docker 镜像，耗时约 5-15 分钟。后续启动通常在 30-60 秒内完成。

### 2.4 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 统一网关 | `http://localhost:3000` | Nginx 反向代理入口 |
| 后端 API | `http://localhost:8080` | Drogon REST API |
| 管理后台 | `http://localhost:5173` | Vue 3 管理面板 |

### 2.5 验证

```bash
./scripts/docker-test.sh
```

该脚本自动执行健康检查、匿名登录、核心接口调用等冒烟测试。

### 2.6 停止

```bash
./scripts/docker-down.sh          # 停止容器，保留数据卷
./scripts/docker-down.sh --purge  # 停止容器并删除数据卷
```

## 3. 本地开发环境部署

适用于需要断点调试、修改代码后快速验证的场景。

### 3.1 安装后端编译依赖

Arch Linux：

```bash
sudo pacman -S base-devel cmake git gcc clang \
  postgresql-libs redis hiredis jsoncpp yaml-cpp \
  brotli c-ares zlib openssl curl
```

Ubuntu：

```bash
sudo apt update
sudo apt install build-essential cmake git g++ \
  libpq-dev redis-server libhiredis-dev libjsoncpp-dev \
  libyaml-cpp-dev libbrotli-dev libc-ares-dev zlib1g-dev \
  libssl-dev libcurl4-openssl-dev
```

### 3.2 安装 Drogon 框架

```bash
git clone https://github.com/drogonframework/drogon.git
cd drogon
git submodule update --init
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(nproc)
sudo make install
```

### 3.3 安装 ONNX Runtime（可选）

启用本地神经网络情感分析模型推理：

```bash
wget https://github.com/microsoft/onnxruntime/releases/download/v1.22.0/onnxruntime-linux-x64-1.22.0.tgz
tar xzf onnxruntime-linux-x64-1.22.0.tgz
sudo cp -r onnxruntime-linux-x64-1.22.0/include/* /usr/local/include/
sudo cp -r onnxruntime-linux-x64-1.22.0/lib/* /usr/local/lib/
sudo ldconfig
```

### 3.4 启动基础服务

确认 PostgreSQL 和 Redis 正在运行：

```bash
sudo systemctl start postgresql redis
ss -ltnp | grep -E ':5432|:6379'
```

### 3.5 启动后端

方式一：统一启动脚本（推荐）

```bash
cd backend
./start.sh
```

`start.sh` 执行流程：
1. 加载 `.env` 环境变量
2. 生成 `config.runtime.json`（Drogon 运行时配置）
3. 检查 PostgreSQL / Redis 连通性
4. 顺序执行 `migrations/*.sql` 数据库迁移
5. 编译后端（设置 `HEARTLAKE_SKIP_BUILD=true` 可跳过）
6. 启动服务并执行健康检查

方式二：手动构建与启动

```bash
cd backend
cp .env.example .env    # 编辑 .env 填入数据库连接信息
mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
make -j$(nproc)
cd ..
./build/HeartLake
```

验证后端启动成功：

```bash
curl -sS http://127.0.0.1:8080/api/health
# 预期响应：{"code":0,"message":"success","data":{"status":"ok"}}
```

停止后端：

```bash
cd backend && ./stop.sh
```

### 3.6 启动管理后台

```bash
cd admin
npm install
npm run dev
# 访问 http://localhost:5173
```

### 3.7 启动 Flutter 移动端

```bash
cd frontend
flutter pub get
flutter run
```

Web 模式运行：

```bash
flutter run -d web-server --web-hostname 0.0.0.0 --web-port 3000
```

### 3.8 启动 Ollama（可选）

```bash
ollama serve
curl -sS http://127.0.0.1:11434/api/tags    # 验证服务可用
```

后端通过 `AI_PROVIDER` / `AI_BASE_URL` / `AI_MODEL` 环境变量连接 Ollama。Ollama 未启动时 AI 对话功能降级为默认回复，EdgeAI 本地情感分析不受影响。

## 4. 环境变量配置

复制 `.env.example` 为 `.env` 后按需修改。Docker 部署下 `DB_HOST` / `REDIS_HOST` 自动指向容器内服务名，无需手动配置。

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DB_HOST` | `localhost` | PostgreSQL 地址 |
| `DB_PORT` | `5432` | PostgreSQL 端口 |
| `DB_NAME` | `heartlake` | 数据库名称 |
| `DB_USER` | `postgres` | 数据库用户名 |
| `DB_PASSWORD` | 必填 | 数据库密码 |
| `REDIS_HOST` | `localhost` | Redis 地址 |
| `REDIS_PORT` | `6379` | Redis 端口 |
| `REDIS_PASSWORD` | 必填 | Redis 认证密码 |
| `PASETO_KEY` | （32 字节默认值） | PASETO v4 用户端签名密钥，长度 >= 32 字节 |
| `ADMIN_PASETO_KEY` | （32 字节默认值） | PASETO v4 管理端签名密钥 |
| `AI_PROVIDER` | `ollama` | AI 推理提供者 |
| `AI_BASE_URL` | `http://localhost:11434` | AI 服务地址 |
| `AI_MODEL` | `heartlake-qwen` | Ollama 模型名称 |
| `AI_TIMEOUT` | `30` | AI 推理超时时间（秒） |
| `SERVER_HOST` | `0.0.0.0` | 后端监听地址 |
| `SERVER_PORT` | `8080` | 后端监听端口 |
| `GATEWAY_PORT` | `3000` | Nginx 网关端口 |
| `ADMIN_PORT` | `5173` | 管理后台端口 |

## 6. 常见问题

### 后端启动后立即退出

| 可能原因 | 排查方法 |
|----------|----------|
| PostgreSQL 不可达 | `pg_isready -h localhost -p 5432` |
| Redis 不可达 | `redis-cli -a "$REDIS_PASSWORD" ping` |
| PASETO_KEY 长度不足 | 检查 `.env`，确保 >= 32 字节 |
| 端口被占用 | `ss -ltnp | grep :8080` |

### 接口返回 500

1. 检查后端日志中的首条异常信息
2. 确认数据库迁移已完整执行：`cd backend && ./start.sh`
3. 验证数据库连通性：`psql -h localhost -U postgres -d heartlake -c "SELECT 1"`

### 前端页面空白

1. 确认后端 8080 端口正在监听
2. 检查浏览器控制台是否有网络请求错误
3. 强制刷新页面：`Ctrl + Shift + R`（清除 Service Worker 缓存）

### AI 对话功能降级

- Ollama 未启动或模型未加载时，AI 对话返回降级回复
- EdgeAI 本地情感分析不依赖 Ollama，始终可用
- 验证 Ollama 状态：`curl -sS http://127.0.0.1:11434/api/tags`

### Docker 容器启动异常

```bash
docker compose ps                          # 查看容器状态
docker compose logs backend --tail=100     # 查看后端日志
docker compose logs postgres --tail=50     # 查看数据库日志
docker compose restart backend             # 重启单个容器
docker compose down && docker compose up -d --build  # 完全重建
```
