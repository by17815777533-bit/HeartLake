# 本地启动与运行手册

## 环境要求

| 组件 | 最低版本 | 用途 |
|------|----------|------|
| Linux | Ubuntu 22.04+ / Arch | 开发宿主机 |
| GCC / Clang | GCC 12+ / Clang 15+ | C++20 编译 |
| CMake | 3.22+ | 构建系统 |
| PostgreSQL | 16 | 主数据库 |
| Redis | 7 | 缓存 + 会话 |
| Node.js | 18+ | 管理后台构建 |
| Flutter | 3.19+ (Dart 3.3+) | 移动端开发 |
| Docker + Compose | 24+ / v2 | 容器化部署 |
| Ollama | latest | 本地 LLM 推理（可选） |

## 方式一：Docker 一键启动（推荐）

最快的方式，6 个容器自动编排，无需手动安装任何依赖。

```bash
cd heartlake

# 复制环境变量模板
cp .env.example .env

# 一键启动全部服务
./scripts/docker-up.sh

# 验证
./scripts/docker-test.sh

# 关闭
./scripts/docker-down.sh
```

`docker-compose.yml` 编排了以下服务：

| 服务 | 镜像 | 端口 | 健康检查 |
|------|------|------|----------|
| postgres | postgres:16 | 5432 | `pg_isready` |
| redis | redis:7 | 6379 | `redis-cli ping` |
| ollama | ollama/ollama | 11434 | - |
| backend | 自构建 (Drogon) | 8080 | `curl /api/health` |
| admin | 自构建 (Nginx + Vue) | 5173 | - |
| gateway | nginx:1.27-alpine | 3000 | - |

启动顺序由 `depends_on` + `condition: service_healthy` 控制：
1. PostgreSQL 和 Redis 先启动，通过健康检查后
2. Backend 启动，自动执行数据库迁移
3. Admin 和 Gateway 最后启动

启动后访问：
- 统一网关: http://localhost:3000
- 后端 API: http://localhost:8080
- 管理后台: http://localhost:5173

## 方式二：本地开发启动

适合需要调试源码、附加断点的场景。

### 1. 安装后端依赖

Arch Linux:
```bash
sudo pacman -S base-devel cmake git gcc clang \
  postgresql-libs redis hiredis jsoncpp yaml-cpp \
  brotli c-ares zlib openssl curl
```

Ubuntu:
```bash
sudo apt install build-essential cmake git g++ \
  libpq-dev redis-server libhiredis-dev libjsoncpp-dev \
  libyaml-cpp-dev libbrotli-dev libc-ares-dev zlib1g-dev \
  libssl-dev libcurl4-openssl-dev
```

安装 Drogon 框架:
```bash
git clone https://github.com/drogonframework/drogon.git
cd drogon && git submodule update --init
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
make -j$(nproc) && sudo make install
```

ONNX Runtime（可选，启用本地神经网络情感分析）:
```bash
wget https://github.com/microsoft/onnxruntime/releases/download/v1.22.0/onnxruntime-linux-x64-1.22.0.tgz
tar xzf onnxruntime-linux-x64-1.22.0.tgz
sudo cp -r onnxruntime-linux-x64-1.22.0/include/* /usr/local/include/
sudo cp -r onnxruntime-linux-x64-1.22.0/lib/* /usr/local/lib/
sudo ldconfig
```

### 2. 启动基础服务

确保 PostgreSQL 和 Redis 正在运行：
```bash
# 检查端口占用
ss -ltnp | grep -E ':5432|:6379|:8080|:11434'

# 如果用 systemd 管理
sudo systemctl start postgresql redis
```

### 3. 启动后端

```bash
cd backend

# 方式一：统一启动脚本（自动迁移 + 构建 + 启动）
./start.sh

# 方式二：手动构建
mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
make -j$(nproc)
cd .. && ./build/HeartLake
```

`start.sh` 的工作流程：
1. 加载 `.env` 环境变量
2. 生成 `config.runtime.json`（Drogon 运行时配置）
3. 检查 PostgreSQL / Redis 可达性
4. 顺序执行 `migrations/*.sql` 确保 schema 与代码对齐
5. 构建后端二进制（可通过 `HEARTLAKE_SKIP_BUILD=true` 跳过）
6. 启动服务并做健康检查

健康检查：
```bash
curl -sS http://127.0.0.1:8080/api/health
# 预期返回: {"status":"ok", ...}
```

### 4. 启动管理后台

```bash
cd admin
npm install
npm run dev
# 访问 http://localhost:5173
```

### 5. 启动 Flutter 前端

```bash
cd frontend
flutter pub get
flutter run -d web-server --web-hostname 0.0.0.0 --web-port 3000
# 访问 http://localhost:3000
```

### 6. 启动 Ollama（可选）

```bash
ollama serve
# 验证: curl -sS http://127.0.0.1:11434/api/tags
```

后端通过 `AI_PROVIDER` / `AI_BASE_URL` / `AI_MODEL` 环境变量连接 Ollama。模型服务不可达时，AI 相关功能会降级但系统仍可正常运行。

## 环境变量说明

复制 `.env.example` 为 `.env`，关键配置项：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DB_HOST` | localhost | PostgreSQL 地址 |
| `DB_PORT` | 5432 | PostgreSQL 端口 |
| `DB_NAME` | heartlake | 数据库名 |
| `DB_USER` | postgres | 数据库用户 |
| `DB_PASSWORD` | HeartLake | 数据库密码 |
| `REDIS_HOST` | localhost | Redis 地址 |
| `REDIS_PORT` | 6379 | Redis 端口 |
| `REDIS_PASSWORD` | HeartLake | Redis 密码 |
| `PASETO_KEY` | (32字节) | PASETO v4 令牌签名密钥，长度必须 >= 32 字节 |
| `ADMIN_PASETO_KEY` | (32字节) | 管理后台独立签名密钥 |
| `AI_PROVIDER` | ollama | AI 推理提供者 |
| `AI_BASE_URL` | http://localhost:11434 | AI 服务地址 |
| `AI_MODEL` | heartlake-qwen | 使用的模型名称 |
| `SERVER_HOST` | 0.0.0.0 | 后端监听地址 |
| `SERVER_PORT` | 8080 | 后端监听端口 |

Docker 环境下，`DB_HOST` / `REDIS_HOST` 等会自动指向容器内服务名（postgres / redis）。

## 启动后验证

快速冒烟测试：
```bash
# 1. 健康检查
curl -fsS http://127.0.0.1:8080/api/health

# 2. 匿名登录
curl -fsS -X POST http://127.0.0.1:8080/api/auth/anonymous \
  -H 'Content-Type: application/json' \
  -d '{"device_id":"smoke_test"}'

# 3. 管理后台登录
curl -fsS -X POST http://127.0.0.1:8080/api/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"HeartLake"}'
```

全量端到端测试：
```bash
./tests/clients/run_all.sh
```

## 常见问题

### 后端启动失败

症状: 启动后立即退出
- 检查 PostgreSQL 是否可达: `pg_isready -h localhost -p 5432`
- 检查 Redis 是否可达: `redis-cli -a HeartLake ping`
- 检查 PASETO_KEY 长度是否 >= 32 字节
- 查看启动日志中的首条 ERROR

### 接口返回 500

- 大概率是数据库 schema 不完整，重新执行迁移:
  ```bash
  cd backend && ./start.sh  # 会自动执行迁移
  ```
- 手动验证数据库读写: `psql -h localhost -U postgres -d heartlake -c "SELECT 1"`

### 前端白屏

排查顺序：
1. 后端 8080 端口是否在监听
2. 浏览器控制台是否有网络错误
3. 强刷缓存 `Ctrl + Shift + R`（Flutter Web 的 Service Worker 可能缓存了旧版本）
4. 重启 Flutter Web 服务

### AI 功能降级

- Ollama 未启动或模型未拉取时，AI 对话、摘要等功能会返回降级回复
- 本地情感分析（EdgeAI 引擎）不依赖 Ollama，始终可用
- 检查 Ollama: `curl -sS http://127.0.0.1:11434/api/tags`

### Docker 容器启动顺序问题

- Backend 容器依赖 PostgreSQL 和 Redis 的健康检查通过后才启动
- 如果 Backend 反复重启，检查 `docker compose logs postgres` 确认数据库是否就绪
- 首次启动可能需要等待 30-60 秒让所有健康检查通过

## 关闭服务

Docker 方式：
```bash
./scripts/docker-down.sh
# 或
docker compose down
```

本地方式：
```bash
# 后端
cd backend && ./stop.sh
# 或
pkill -f HeartLake

# 前端和管理后台
# Ctrl+C 终止对应进程
```
