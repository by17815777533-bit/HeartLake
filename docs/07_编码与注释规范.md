# 编码与注释规范

本文只描述当前仓库采用的工程约束。

## 1. 总原则

- 先保证契约正确，再做界面和提示。
- 失败必须显式暴露。
- 单一状态源，不允许多处各自维护一套真相。
- 已停用能力直接删掉，不保留分叉和假兜底。

## 2. C++ 后端

- 控制器只做鉴权、参数校验和响应封装。
- 业务逻辑放在 application / service。
- 不写空 `catch`，不写 `catch (...)`。
- 不用默认值拼装假成功。
- 注释解释边界和原因，不解释代码表面动作。

## 3. Flutter 客户端

- datasource 负责协议归一和抛错。
- provider 负责状态编排。
- 页面只消费状态，不自己猜协议。
- 空态、错误态、warning 态必须可见。

## 4. Vue 管理端

- API 调用集中在 `src/api`、`src/services`。
- 列表与分页统一走归一化工具。
- 图表和统计失败要保留 stale 状态。

## 5. 文档与注释

- 文档只写现状。
- 代码注释只写当前职责和当前边界。
- 不写迁移史，不写无关背景，不写过时口径。
