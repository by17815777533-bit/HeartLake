## 一、项目概述

HeartLake（心湖）是一个匿名情感社交平台，用户将心事写成「心石」投入湖中，系统通过纯 C++ 边缘 AI 引擎进行情感分析与同频共鸣匹配，在端到端加密和差分隐私保护下实现完全匿名的情感连接。

技术栈：C++20 + Drogon（后端）、Flutter（移动端）、Vue 3 + Element Plus（管理后台）。

代码量：75,000+ 行，零第三方 AI 云服务依赖。

## 二、核心技术创新

### 2.1 边缘 AI 引擎（8 大子系统，纯 C++ 实现）

将传统需要云端 GPU 的 AI 能力完全下沉到后端进程内，实现毫秒级推理和零数据外泄。

| 子系统 | 算法 | 延迟 |
|--------|------|------|
| 情感分析 | 规则引擎 + 情感词典(12000+词) + ONNX Roberta | <5ms / <50ms |
| 内容审核 | Aho-Corasick 多模式匹配 + 心理风险评估 | <1ms |
| HNSW 向量索引 | Hierarchical Navigable Small World, 128维 | <10ms |
| 联邦学习 | FedAvg 加权聚合 + stale-aware 异步衰减 | 异步 |
| 差分隐私 | Laplace 机制 + Gaussian 噪声 + 隐私预算追踪 | <0.1ms |
| 情感共鸣 | Dynamic Time Warping + 时间衰减 + 多样性惩罚 | <20ms |
| 双记忆 RAG | 短期记忆(5条) + 长期记忆(30天) + 本地大模型 | <500ms |
| 推荐引擎 | 协同过滤 + 内容相似 + Thompson Sampling 探索 | <30ms |

架构采用门面模式，EdgeAIEngine 作为统一调度入口，8 个子系统独立编译、独立测试。

### 2.2 隐私计算体系

三层隐私保护，超越 GDPR 标准：

- 差分隐私：Laplace 机制为统计查询注入校准噪声，epsilon 预算全局追踪，防止差分攻击
- 联邦学习：FedAvg 加权聚合，模型参数在本地训练，仅上传梯度，原始数据不出端
- 身份影子映射：用户真实身份与社交身份完全隔离，即使数据库泄露也无法关联

### 2.3 三层情感融合分析

```
输入文本 → 规则引擎(否定/程度/转折) → 情感词典(12000+词条) → ONNX 统计模型
                                                                    ↓
                                                              加权融合决策
                                                                    ↓
                                                        mood + score + confidence
```

- 规则层：处理否定词、程度副词、转折连词，覆盖中文语法特殊结构
- 词典层：12000+ 中文情感词条，支持领域自适应权重
- 模型层：ONNX Runtime 加载 Roberta 模型，离线推理无需网络

验证指标（32 条中文测试集）：
- Mood Top-1 准确率：96.88%
- Macro-F1：97.69%
- 极性准确率：96.88%

### 2.4 安全架构

| 层级 | 技术 | 说明 |
|------|------|------|
| 认证 | PASETO v4 | 替代 JWT，消除算法混淆攻击面 |
| 传输 | X25519 + AES-256-GCM | 心理咨询端到端加密，前向保密 |
| 存储 | Argon2id | 密码哈希，抗 GPU/ASIC 暴力破解 |
| 防护 | 三级限流 + CORS + CSP | 令牌桶/滑动窗口/固定窗口 |
| 审计 | 安全审计日志 + 风险评分 | 实时监控异常行为 |

### 2.5 情感共鸣匹配

基于 Dynamic Time Warping 的情感时间序列匹配算法：

1. 提取用户近期情感轨迹（mood sequence）
2. DTW 计算轨迹相似度，容忍时间轴伸缩
3. 时间衰减因子：近期互动权重更高
4. 多样性惩罚：避免推荐同质化内容
5. 结合 HNSW 向量索引加速候选检索

### 2.6 多视角亲密分算法

四维度关系量化：

- 互动频率：消息/涟漪/纸船的交互次数
- 情感共鸣：DTW 相似度得分
- 时间衰减：指数衰减，半衰期可配置
- 互惠度：双向互动的平衡性

## 三、架构设计

### 3.1 整体架构

```
Flutter Mobile ──┐                    ┌── Vue 3 Admin
                 ├── Nginx Reverse ───┤
                 │      Proxy         │
                 └────────────────────┘
                          │
                 ┌────────┴────────┐
                 │  C++20 Drogon   │
                 │  Async Backend  │
                 ├─────────────────┤
                 │  EdgeAIEngine   │ ← 8 子系统
                 │  DDD Layers     │ ← 领域/应用/基础设施/接口
                 │  ServiceLocator │ ← 依赖注入
                 │  Event Bus      │ ← 事件驱动
                 ├─────────────────┤
                 │ PostgreSQL 16   │
                 │ Redis 7         │
                 └─────────────────┘
```

### 3.2 DDD 分层

| 层 | 职责 | 示例 |
|----|------|------|
| interfaces/api | HTTP 控制器，请求/响应转换 | StoneController, UserController |
| application | 应用服务，编排领域逻辑 | StoneApplicationService |
| domain | 领域模型，业务规则 | Stone, User, Friendship |
| infrastructure | 基础设施，数据库/缓存/AI | Repository, RedisCache, EdgeAIEngine |

### 3.3 WebSocket 实时通信

- 心跳检测 + 自动重连
- 房间广播机制（聊天室/通知）
- 并发安全：shared_mutex 保护连接池
- 消息压缩 + 批量发送优化
