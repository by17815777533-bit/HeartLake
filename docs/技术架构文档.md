## 1. 系统架构总览

HeartLake（心湖）是一个面向情感社交的全栈平台，采用 **C++20 高性能后端 + Flutter 跨平台移动端 + Vue 3 管理后台** 的三端架构，内置自研 EdgeAI 引擎实现端侧智能。

### 1.1 技术栈全景

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 移动端 | Flutter / Dart | 3.x |
| 管理后台 | Vue 3 + Vite + Element Plus | Vue 3.4+ |
| 后端框架 | Drogon (C++20) | 1.9+ |
| 数据库 | PostgreSQL | 16-alpine |
| 缓存 | Redis | 7-alpine |
| AI 推理 | Ollama (heartlake-qwen) + ONNX Runtime | 0.6.2 |
| 反向代理 | Nginx | 1.27-alpine |
| 容器编排 | Docker Compose | v2 |

### 1.3 核心设计原则

- **DDD 驱动**: 领域模型为核心，业务逻辑不泄漏到基础设施层
- **事件驱动**: EventBus 解耦领域事件，异步处理 AI 分析任务
- **隐私优先**: 差分隐私 + 端到端加密 + 身份影子映射，三重保护
- **边缘智能**: 8 大 AI 子系统本地运行，零数据外传

### 2.2 各层详解

#### 接口层 (20 个 Controller)

负责 HTTP/WebSocket 请求的接收、参数校验、响应序列化。不包含任何业务逻辑。

| Controller | 职责 |

| UserController | 用户信息管理 |
| AccountController | 注册 / 登录 / Token 刷新 |
| FriendController | 好友关系管理 |
| InteractionController | 涟漪（点赞）/ 评论 |
| PaperBoatController | 纸船（私信）收发 |
| EdgeAIController | AI 能力 HTTP 端点 |
| RecommendationController | 推荐接口 |
| AdminController | 管理后台接口 |
| BroadcastWebSocketController | WebSocket 实时推送 |
| GuardianController | 心理守护接口 |
| SafeHarborController | 安全港湾接口 |
| PrivacyController | 隐私设置接口 |
| VectorSearchController | 向量搜索接口 |
| ConsultationController | 咨询接口 |
| ReportController | 举报接口 |
| VIPController | VIP 功能接口 |
| AdminManagementController | 管理员管理 |
| TempFriendController | 临时好友接口 |
| HealthController | 健康检查 |

#### 应用层 (ApplicationService)

编排用例流程，协调领域服务、仓储、缓存和事件。以 `StoneApplicationService` 为例：

```cpp
class StoneApplicationService {
private:
    std::shared_ptr<domain::stone::StoneService> stoneService_;      // 领域服务
    std::shared_ptr<domain::stone::IStoneRepository> stoneRepository_; // 仓储接口
    std::shared_ptr<core::cache::CacheManager> cacheManager_;         // 缓存管理
    std::shared_ptr<core::events::EventBus> eventBus_;                // 事件总线

public:
    // 发布石头：校验 → 领域逻辑 → 持久化 → 发布事件 → 异步AI处理
    Json::Value publishStone(userId, content, stoneType, stoneColor, moodType, isAnonymous);
    // 异步处理：情感分析 + 内容审核 + 向量索引
    void processStoneAsync(stoneId, userId, content, moodType);
};
```

**ApplicationServiceFactory** 通过静态 `initialize()` 方法将所有应用服务注册到 DI 容器。

#### 领域层 (Domain Aggregates)

纯业务逻辑，不依赖任何基础设施。通过接口（`IStoneRepository`、`IFriendRepository`）实现依赖倒置。

```cpp
// 领域实体
struct StoneEntity {
    std::string stoneId, userId, content, stoneType, stoneColor, moodType, status, createdAt;
    bool isAnonymous;
    int ripplesCount, boatsCount, viewsCount;
};

// 仓储接口（领域层定义，基础设施层实现）
class IStoneRepository {
    virtual StoneEntity save(const StoneEntity& stone) = 0;
    virtual std::optional<StoneEntity> findById(const std::string& stoneId) = 0;
    // ...
};
```

#### 基础设施层

| 组件 | 实现 | 说明 |
|------|------|------|
| ServiceLocator | `std::type_index` + `recursive_mutex` | 类型擦除 DI 容器，支持单例/实例注册 |
| EventBus | 类型索引 + `std::any` 处理器 | 快照式发布避免死锁 |
| CacheManager | Redis 7 客户端封装 | 256MB allkeys-lru 策略 |
| EdgeAIEngine | 8 大子系统 | 详见第 3 章 |
| Repository 实现 | PostgreSQL 16 | 实现领域层仓储接口 |

### 2.3 依赖注入与服务定位

```mermaid
sequenceDiagram
    participant Main as main()
    participant ASF as ApplicationServiceFactory
    participant SL as ServiceLocator
    participant SAS as StoneApplicationService

    Main->>ASF: initialize()
    ASF->>SL: registerSingleton<StoneService>(...)
    ASF->>SL: registerSingleton<IStoneRepository>(...)
    ASF->>SL: registerSingleton<CacheManager>(...)
    ASF->>SL: registerSingleton<EventBus>(...)
    ASF->>SL: registerInstance<StoneApplicationService>(...)
    Note over SL: 所有服务注册完毕

    Main->>SL: resolveRequired<StoneApplicationService>()
    SL-->>Main: shared_ptr<StoneApplicationService>
    Main->>SAS: publishStone(...)
```

ServiceLocator 采用 `recursive_mutex` 保证线程安全，支持三种注册模式：

- `registerInstance<T>`: 注册已有实例
- `registerSingleton<T>`: 延迟创建单例
- `resolve<T>` / `resolveRequired<T>`: 解析服务（后者在未找到时抛异常）

### 3.2 子系统详解

#### ① 本地情感分析 — 三层融合引擎

采用规则层 + 词典层 + 统计层的三层融合架构，无需网络即可完成情感分析。

```mermaid
graph LR
    Input["输入文本"] --> R["规则层<br/>正则模式匹配"]
    Input --> L["词典层<br/>情感词典查询"]
    Input --> S["统计层<br/>ONNX 模型推理"]

    R --> Fusion["融合决策"]
    L --> Fusion
    S --> Fusion
    Fusion --> Output["情感标签 + 置信度"]
```

- **规则层**: 基于正则表达式的快速模式匹配，处理明确的情感表达
- **词典层**: 内置中文情感词典，支持程度副词、否定词、转折词的组合计算
- **统计层**: ONNX Runtime 加载 `sentiment_zh.onnx` 模型，INT8 量化推理
- **融合策略**: 三层结果加权融合，规则层优先级最高（处理极端情况）

#### ② 内容审核 — AC 自动机 + 五因子心理风险

```mermaid
graph TB
    Input["用户内容"] --> Norm["文本规范化<br/>移除干扰字符"]
    Norm --> BF["计数布隆过滤器<br/>快速预检"]
    BF -->|可能命中| AC["AC 自动机<br/>O(n) 多模式匹配"]
    BF -->|确定不含| Pass["直接放行"]
    AC --> WL["白名单过滤"]
    WL --> Risk["五因子心理风险评估"]
    Risk --> Cache["分片 LRU 缓存<br/>热点结果缓存"]
    Cache --> Result["审核结果"]
```

**五因子心理风险模型**:

| 因子 | 说明 |
|------|------|
| selfHarmIntent | 自伤意图检测 |
| hopelessness | 绝望感评估 |
| socialIsolation | 社交孤立程度 |
| temporalUrgency | 时间紧迫性 |
| linguisticMarkers | 语言学标记 |

**ContentFilter 核心组件**:

- `ACAutomaton`: Aho-Corasick 自动机，O(n) 时间复杂度多模式匹配
- `CountingBloomFilter<>`: 计数布隆过滤器，支持删除操作的概率预检
- `ShardedLRUCache<string, string>`: 分片 LRU 缓存，减少锁竞争
- `reload()`: 支持热更新敏感词库，无需重启服务

#### ③ 情感脉搏检测

基于滑动窗口的情感趋势分析，使用 MAD（中位数绝对偏差）进行异常值剔除。

- 滑动窗口收集用户近期情感数据
- MAD 算法剔除异常波动点
- 检测情感突变（如突然从积极转为消极）
- 触发心理守护预警机制

#### ④ 联邦学习 — FedAvg + DP-SGD

```mermaid

    participant Node2 as 边缘节点 2

    Node2->>Node2: 本地训练 (DP-SGD)

    Note over Node2: 梯度裁剪 + 高斯噪声

    Node1->>Server: 上传加噪梯度
    Node2->>Server: 上传加噪梯度

    Server->>Server: FedAvg 加权聚合
    Server-->>Node1: 下发全局模型
    Server-->>Node2: 下发全局模型
```

- **FedAvg**: 按数据量加权聚合各节点模型参数
- **DP-SGD**: 梯度裁剪 + 高斯噪声注入，保证训练过程的差分隐私
- 原始数据永远不离开本地节点

#### ⑤ 差分隐私 — Laplace 机制

```cpp

struct PrivateEmotionStats {
```

- **Laplace 噪声**: 对情感统计数据添加 Laplace 分布噪声

- **7 种情绪类别**: 覆盖完整情感谱系
- 理论基础: FedMultiEmo 论文

## 4. 隐私保护洋葱模型

HeartLake 采用四层洋葱式隐私保护架构，从内到外层层加固。

### 4.1 洋葱模型图

```mermaid
graph TB
    subgraph Layer4["第四层：传输安全"]
        TLS["TLS 1.2/1.3<br/>ECDHE 密钥交换"]
        subgraph Layer3["第三层：端到端加密"]
            E2E["X25519 ECDH + AES-256-GCM<br/>IND-CCA2 安全性 · 前向保密"]
            subgraph Layer2["第二层：身份保护"]
                Shadow["身份影子映射<br/>HMAC-SHA256 单向哈希<br/>影子 ID 轮换"]
                subgraph Layer1["第一层：数据隐私"]
                    DP["差分隐私<br/>Laplace 噪声<br/>ε-预算追踪"]
                    FL["联邦学习<br/>DP-SGD<br/>数据不出本地"]
                end
            end
        end
    end
```

### 4.2 各层详解

#### 第一层：数据隐私（差分隐私 + 联邦学习）

- 情感统计数据添加 Laplace 噪声后才可查询
- 联邦学习确保原始数据不离开本地
- DP-SGD 在训练过程中注入高斯噪声

#### 第二层：身份保护（影子映射）

```cpp
// IdentityShadowMap.h
struct ShadowIdentity {
    std::string shadowId;        // HMAC-SHA256 生成的影子 ID
    std::string hashedIp;        // IP 地址匿名化
    std::string hashedFingerprint; // 设备指纹匿名化
};
```

- **HMAC-SHA256 单向哈希**: 真实用户 ID → 不可逆的影子 ID
- **影子 ID 轮换**: 定期更换影子 ID，防止长期追踪
- **IP / 指纹匿名化**: 所有标识信息单向哈希处理

#### 第三层：端到端加密

```mermaid
sequenceDiagram
    participant Alice as 用户 A
    participant Server as 服务器
    participant Bob as 用户 B

    Alice->>Alice: 生成 X25519 密钥对
    Bob->>Bob: 生成 X25519 密钥对

    Alice->>Server: 上传公钥
    Bob->>Server: 上传公钥

    Alice->>Server: 获取 Bob 公钥
    Alice->>Alice: ECDH 协商 → HKDF-SHA256 派生密钥
    Alice->>Alice: AES-256-GCM 加密消息

    Alice->>Server: 发送密文
    Server->>Bob: 转发密文（服务器无法解密）

    Bob->>Bob: ECDH 协商 → 相同密钥
    Bob->>Bob: AES-256-GCM 解密
```

| 参数 | 值 | 说明 |
|------|-----|------|
| KEY_SIZE | 32 字节 | AES-256 密钥长度 |
| IV_SIZE | 12 字节 | GCM 随机数长度 |
| TAG_SIZE | 16 字节 | 认证标签长度 |
| 密钥交换 | X25519 ECDH | 椭圆曲线 Diffie-Hellman |
| 密钥派生 | HKDF-SHA256 | 基于 HMAC 的密钥派生 |
| 加密算法 | AES-256-GCM | AEAD 认证加密 |
| 安全性 | IND-CCA2 | 选择密文攻击安全 |
| 前向保密 | ✅ | 临时密钥对，历史消息不可追溯 |

#### 第四层：传输安全

- TLS 1.2 / 1.3 强制加密
- ECDHE 密钥交换，支持前向保密
- HSTS 头部: `max-age=31536000; includeSubDomains; preload`

### 5.1 安全架构总览图

```
熵值: log2(256^6) = 48 bits
```

相比传统英文助记词，中文恢复密钥对中文用户更友好、更易记忆。

### 8.3 内容过滤性能

ContentFilter 采用三级加速架构：

```
请求 → Counting Bloom Filter (O(1) 预检)
         ↓ 可能存在
       AC 自动机 (O(n) 精确匹配)
         ↓ 命中
       Sharded LRU Cache (缓存热点结果)
```

| 组件 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| Bloom Filter | O(k) ≈ O(1) | O(m) | k 个哈希函数，m 位数组 |
| AC 自动机 | O(n + z) | O(Σ\|p\|) | n=文本长度，z=匹配数 |
| Sharded LRU | O(1) 均摊 | O(capacity) | 分片降低锁竞争 |
| 白名单 | O(1) | O(w) | unordered_set 查找 |
