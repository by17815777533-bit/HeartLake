# 端到端测试与故障排查手册

## 测试体系概览

HeartLake 的测试覆盖三个层级：

| 层级 | 工具 | 测试数 | 覆盖范围 |
|------|------|--------|----------|
| 后端单元测试 | Google Test | 55+ | AI 引擎、工具类、领域逻辑、并发安全 |
| 前端单元测试 | Dart test | 33 文件 | 模型序列化、EdgeAI、服务层 |
| 管理后台测试 | Vitest | 25 文件 | 组件渲染、API 调用、Store 状态、压力测试 |
| 端到端测试 | docker-test.sh | - | 全链路冒烟（Docker 环境） |

## 运行测试

### 后端测试

```bash
cd backend/build
cmake -DCMAKE_BUILD_TYPE=Debug ..
make -j$(nproc)
ctest --output-on-failure -j8
```

测试套件包括：
- `test_edge_ai` — EdgeAI 引擎 8 子系统功能验证
- `test_sentiment` — 情感分析准确率（中英文）
- `test_hnsw` — HNSW 向量检索正确性和性能
- `test_differential_privacy` — 差分隐私噪声分布和预算追踪
- `test_federated` — 联邦学习聚合正确性
- `test_content_filter` — 内容过滤 AC 自动机匹配
- `test_e2e_encryption` — X25519 + AES-256-GCM 加解密
- `test_circuit_breaker` — 熔断器状态机转换
- `test_validator` — 输入校验
- `benchmark_edge_ai` — AI 引擎性能基准
- `test_concurrency_torture` — 高并发压力测试
- `test_ai_torture` — AI 引擎极限压力测试

ASAN 构建（内存安全验证）：
```bash
mkdir -p build-asan && cd build-asan
cmake -DCMAKE_BUILD_TYPE=Debug \
  -DCMAKE_CXX_FLAGS="-fsanitize=address -fno-omit-frame-pointer" ..
make -j$(nproc)
ctest --output-on-failure
```

### 前端测试

```bash
cd frontend
flutter test
```

关键测试文件：
- `test/edge_ai_test.dart` — 端侧情感分类器和本地差分隐私
- `test/models/` — 实体序列化/反序列化
- `test/stress/` — 压力测试

### 管理后台测试

```bash
cd admin
npx vitest run
```

测试覆盖：
- `__tests__/views/` — 每个页面组件的渲染和交互测试
- `__tests__/composables/` — 可组合函数逻辑测试
- `__tests__/stress/` — 大数据量下的 API / Dashboard / Store 压力测试
- `__tests__/websocket.test.ts` — WebSocket 连接和消息处理

### Docker 端到端测试

```bash
cd heartlake
./scripts/docker-test.sh
```

这个脚本会：
1. 启动全部 Docker 容器
2. 等待健康检查通过
3. 执行冒烟测试（健康检查、匿名登录、核心接口调用）
4. 在容器内运行后端 / 前端 / 管理后台的完整测试套件
5. 输出汇总结果

## 核心链路验证用例

### 用例 A：认证 → 发布 → 互动

```bash
# 1. 匿名登录
TOKEN=$(curl -sS -X POST http://localhost:8080/api/auth/anonymous \
  -H 'Content-Type: application/json' \
  -d '{"device_id":"e2e_test"}' | jq -r '.data.token')

# 2. 发布石头
curl -sS -X POST http://localhost:8080/api/stones \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"今天心情不错","mood_type":"happy"}'

# 3. 获取心湖列表
curl -sS http://localhost:8080/api/lake/stones?page=1&page_size=10 \
  -H "Authorization: Bearer $TOKEN"

# 4. 点涟漪（幂等）
curl -sS -X POST http://localhost:8080/api/stones/{stone_id}/ripples \
  -H "Authorization: Bearer $TOKEN"
```

验收标准：全部返回 `code: 0`，涟漪重复点击返回 `already_rippled: true`。

### 用例 B：好友与消息

```bash
# 1. 好友列表
curl -sS http://localhost:8080/api/friends \
  -H "Authorization: Bearer $TOKEN"

# 2. 发送消息（需亲密度 >= 12）
curl -sS -X POST http://localhost:8080/api/friends/{friend_id}/messages \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"你好"}'

# 3. 删除好友（可恢复隐藏）
curl -sS -X DELETE http://localhost:8080/api/friends/{friend_id} \
  -H "Authorization: Bearer $TOKEN"
# 预期返回 mode: intimacy_auto_hidden

# 4. 删除后发消息应返回 403
curl -sS -X POST http://localhost:8080/api/friends/{friend_id}/messages \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"test"}'
# 预期返回 code: 403
```

### 用例 C：情绪追踪

```bash
# 1. 情绪日历
curl -sS "http://localhost:8080/api/users/my/emotion-calendar?year=2026&month=2" \
  -H "Authorization: Bearer $TOKEN"

# 2. 情绪热力图
curl -sS http://localhost:8080/api/users/my/emotion-heatmap \
  -H "Authorization: Bearer $TOKEN"

# 3. 情绪趋势
curl -sS http://localhost:8080/api/recommendations/emotion-trends \
  -H "Authorization: Bearer $TOKEN"

# 4. 实时情绪脉搏
curl -sS http://localhost:8080/api/edge-ai/emotion-pulse \
  -H "Authorization: Bearer $TOKEN"
```

### 用例 D：安全功能

```bash
# 1. 无 token 访问应返回 401
curl -sS http://localhost:8080/api/account/info
# 预期: 401

# 2. 伪造 token 应返回 401
curl -sS http://localhost:8080/api/account/info \
  -H "Authorization: Bearer fake_token_xxx"
# 预期: 401

# 3. 普通用户访问管理接口应返回 403
curl -sS http://localhost:8080/api/admin/dashboard \
  -H "Authorization: Bearer $TOKEN"
# 预期: 403

# 4. 内容审核（敏感词检测）
curl -sS -X POST http://localhost:8080/api/edge-ai/analyze \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"text":"正常的文本内容"}'
```

## 故障排查

### 后端启动失败

| 症状 | 可能原因 | 排查方法 |
|------|----------|----------|
| 启动后立即退出 | PostgreSQL 不可达 | `pg_isready -h localhost` |
| 启动后立即退出 | Redis 不可达 | `redis-cli ping` |
| 启动后立即退出 | PASETO_KEY 长度不足 | 检查 .env 中密钥 >= 32 字节 |
| 端口已被占用 | 上次进程未清理 | `ss -ltnp \| grep 8080` |

### 接口返回 500

排查顺序：
1. 查看后端日志中的首条异常栈（通常是 schema 缺失）
2. 确认数据库迁移已完整执行
3. 检查请求体字段类型是否匹配

```bash
# 查看后端日志
docker compose logs backend --tail=50
# 或本地日志
tail -50 backend/logs/heartlake.log
```

### 前端白屏

排查顺序：
1. 确认后端 8080 端口在监听
2. 浏览器控制台查看网络错误
3. 强刷 `Ctrl + Shift + R` 清除 Service Worker 缓存
4. 重启 Flutter Web 服务

### WebSocket 断连

排查顺序：
1. 连接 URL 是否携带了有效 token
2. 服务端日志是否有鉴权失败记录
3. 心跳是否正常（ping/pong）
4. 网络是否有代理/防火墙拦截 WebSocket 升级

### Docker 容器问题

```bash
# 查看容器状态
docker compose ps

# 查看特定服务日志
docker compose logs backend --tail=100
docker compose logs postgres --tail=50

# 重启单个服务
docker compose restart backend

# 完全重建
docker compose down && docker compose up -d --build
```

### 数据库问题

```bash
# 验证连接
psql -h localhost -U postgres -d heartlake -c "SELECT 1"

# 检查表结构
psql -h localhost -U postgres -d heartlake -c "\dt"

# 重新执行迁移
cd backend && ./start.sh  # 自动执行 migrations/*.sql
```

## 回归测试策略

每次涉及以下模块的改动，至少重跑对应测试：

| 改动范围 | 必跑测试 |
|----------|----------|
| 认证 / 过滤器 | 全部后端测试 + 安全用例 |
| AI 引擎 | `test_edge_ai` + `benchmark_edge_ai` |
| 数据库 schema | 全部后端测试 + 端到端冒烟 |
| WebSocket | `websocket.test.ts` + 实时事件用例 |
| 前端页面 | `flutter test` + 对应页面手动验证 |
| 管理后台 | `npx vitest run` + 对应页面手动验证 |
