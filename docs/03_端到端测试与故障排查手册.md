## 4. 关键故障与处理方法

### 4.1 白屏 / 卡在 DDC 脚本加载
表现：
- 控制台停留在 `DDC is about to load ...`

排查步骤：
1. 确认 `3000` 与 `8080` 端口同时在线。
2. 强刷页面清缓存（`Ctrl + Shift + R`）。
3. 检查 Service Worker 是否被旧版本残留干扰。
4. 重启 Flutter web server。

处置原则：
- 优先排“服务未启动”和“缓存脏数据”，不要直接判定业务逻辑异常。

### 4.2 接口 500
表现：
- 如 `/account/privacy`、`/recommendations/similar-stones/*` 返回 500。

排查步骤：
1. 查后端日志的第一条异常栈。
2. 检查数据库迁移是否完整应用。
3. 校验请求体字段与类型。

处置原则：
- 先保证 schema 对齐，再处理业务逻辑。

### 4.3 WebSocket 频繁断连重连
表现：
- 重复出现 disconnected/reconnected 日志。

排查步骤：
1. 检查握手是否带 token。
2. 验证服务端是否接受 query 鉴权或首包鉴权。
3. 检查心跳是否正常回 pong。

处置原则：
- 优先修鉴权链路，再调重连参数。

## 7. 全链路 E2E 用例（建议评审现场演示）

### 用例 A：匿名登录到主链路
1. 调 `POST /auth/anonymous` 获取 `token/user_id`。
2. 调 `GET /lake/stones` 验证主列表可读。
3. 调 `POST /stones/{id}/ripples` 验证互动写入。
4. 调 `GET /interactions/my/ripples` 验证个人域读取。

验收标准：
- 全部返回 `2xx`。
- 写后读可看到新增记录或计数变化。

### 用例 B：好友与消息链路
1. 调 `GET /friends` 拉取自动好友列表。
2. 调 `POST /friends/{friend_id}/messages` 发送消息。
3. 调 `GET /friends/{friend_id}/messages` 校验消息可读。
4. 调 `DELETE /friends/{friend_id}` 后再次拉取好友列表，确认已隐藏。
5. 被隐藏后再次发送消息，应返回 `403` 并有明确文案。

验收标准：
- 删除操作后列表收敛，不再“删完又出现”。
- 非可聊状态有明确原因，不出现静默失败。

### 用例 C：情绪日历一致性
1. 发布新石头（带 `mood_type` 或触发 AI 分析）。
2. 打开情绪日历页面加载当月数据。
3. 校验新增日期格子出现，并且情绪指数可展示。

验收标准：
- 当月数据键值能被前端正确识别（`YYYY-MM-DD` 与 `DD` 两种格式兼容）。
- 页面不出现“有数据但格子空白”现象。

## 8. 压测指标解释（评审常问“这些数代表什么”）
- `functional_pass_rate`
  - 定义：功能用例通过率。
  - 意义：是否“能用”。
- `security_pass_rate`
  - 定义：鉴权、越权、基础安全校验通过率。
  - 意义：是否“可上线”。
- `consistency_pass_rate`
  - 定义：缓存与实时状态最终一致比例。
  - 意义：是否“看起来可信”。
- `stress_error_rate_max`
  - 定义：压测阶段最高错误率。
  - 意义：峰值压力下是否雪崩。
- `stress_p95_ms_max`
  - 定义：压测阶段最大 P95 延迟。
  - 意义：最差情况下的可交互性。

## 10. 故障排查速查表
- 现象：消息发送按钮可点但列表不更新
  - 排查：检查 `POST /friends/{id}/messages` 返回码和消息体。
  - 处理：若 `403`，看是否被移除或亲密分不足；若 `500`，看后端日志首条异常。
- 现象：删好友后仍出现
  - 排查：`GET /friends` 是否仍返回同 `user_id`。
  - 处理：确认删除接口是否返回 `mode=intimacy_auto_hidden`。
- 现象：情绪日历空白
  - 排查：`GET /users/my/emotion-calendar` 的 `days` 键格式。
  - 处理：确认前端已启用日期键归一化。
- 现象：接口偶发 502
  - 排查：是否仅网关存活、后端主进程已退出。
  - 处理：先拉起后端主进程，再看网关转发状态。

## 11. 情绪三件套专项回归（新增）

### 11.1 情绪日历
步骤：
1. 发布一条带 `mood_type` 的石头。
2. 打开“情绪日历”页面。
3. 不手动刷新，观察当月格子是否自动更新。

通过标准：
- 发布后短时间内可看到当日情绪变化。
- 不出现“请求成功但格子不变”。

### 11.2 情绪热力图
步骤：
1. 保持页面停留在热力图。
2. 另一个页面发布石头后返回。
3. 检查热力图和洞察文案是否更新。

通过标准：
- 页面通过 `new_stone` 事件触发刷新，无需手动下拉。

### 11.3 情绪趋势
步骤：
1. 进入“情绪趋势”页面。
2. 连续发布 2~3 条不同情绪石头。
3. 观察趋势分布与洞察内容变化。

通过标准：
- 趋势条目和洞察文本出现可见变化。
- 隐私预算/情绪脉搏不受旧缓存影响。

## 12. 好友与消息专项回归（新增）

### 12.1 删除好友
步骤：
1. 在好友列表删除一名好友。
2. 立即刷新好友列表。
3. 再次尝试进入与该好友的私聊。

通过标准：
- 列表不再显示该好友。
- 私聊读写返回明确 `403` 原因，而非通用“网络异常”。

### 12.2 恢复好友显示
步骤：
1. 删除好友后，调用/触发“加好友”动作。
2. 再次拉取好友列表。

通过标准：
- 若亲密分条件满足，好友可恢复显示（`restored_hidden_friend=true`）。

### 12.3 聊天页面容错
检查点：
- `friend_id` 字段缺失或异常时，页面给出明确提示，不崩溃、不白屏。
- 发送失败时乐观消息回滚，输入框内容恢复。

## 13. UI 一致性检查（新增）
- 好友列表页与好友聊天页背景均为湖面风格。
- 夜间/日间主题下文字可读性达标。
- 失败态按钮存在可重试路径，不出现“死页面”。

## 14. 压测前置条件（补充）
- 清理浏览器缓存与 Service Worker，避免静态资源脏读影响结果。

- 压测期间固定模型配置，避免因模型热切换导致指标抖动。
