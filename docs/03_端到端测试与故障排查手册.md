# 端到端测试与故障排查手册

> 当前仓库不再保留三端单测目录，交付前验收以构建校验、Docker 联调冒烟和关键链路手动验证为主。

这份文档偏“联调与排障动作”。如果你要做正式上线决策，请同时看 [10_上线检查清单](./10_上线检查清单.md)。

## 1. 验收入口

| 场景 | 命令 | 说明 |
|------|------|------|
| 2C2G 构建校验 | `./scripts/verify-2c2g.sh` | 后端 Release 构建 + 管理端 build + Flutter analyze |
| Docker 联调冒烟 | `./scripts/docker-test.sh` | 网关、后端、管理端启动后执行核心接口与连通性检查 |
| ONNX 烟雾验证 | `VERIFY_ONNX_SMOKE=1 ./scripts/verify-onnx-smoke.sh` | 可选，验证 ONNX Runtime 模型链路 |

## 2. 核心链路验证

### 2.1 认证 -> 投石 -> 涟漪

```bash
BASE_URL=http://121.41.195.165

TOKEN=$(curl -sS -X POST ${BASE_URL}/api/auth/anonymous \
  -H 'Content-Type: application/json' \
  -d '{"device_id":"e2e_smoke"}' | jq -r '.data.token')

STONE_ID=$(curl -sS -X POST ${BASE_URL}/api/stones \
  -H "Authorization: Bearer ${TOKEN}" \
  -H 'Content-Type: application/json' \
  -d '{"content":"今天的湖面很平静","stone_type":"medium","stone_color":"#7A92A3","mood_type":"calm"}' \
  | jq -r '.data.stone_id')

curl -sS -X POST "${BASE_URL}/api/stones/${STONE_ID}/ripples" \
  -H "Authorization: Bearer ${TOKEN}"
```

验收要点：
- 返回业务成功码
- 石头可在 `/api/lake/stones` 查到
- 重复点涟漪走幂等成功，不出现 500

### 2.2 纸船与好友链路

```bash
curl -sS -X POST "${BASE_URL}/api/stones/${STONE_ID}/boats" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H 'Content-Type: application/json' \
  -d '{"content":"愿你今晚好眠"}'

curl -sS ${BASE_URL}/api/interactions/my/boats \
  -H "Authorization: Bearer ${TOKEN}"

curl -sS ${BASE_URL}/api/friends \
  -H "Authorization: Bearer ${TOKEN}"
```

验收要点：
- 纸船发送后，详情页和“我的纸船”列表都能读到数据
- 好友列表接口稳定返回集合结构，不因空列表或分页字段缺失而崩

### 2.3 账号、头像、情绪追踪

```bash
curl -sS ${BASE_URL}/api/account/info \
  -H "Authorization: Bearer ${TOKEN}"

curl -sS "${BASE_URL}/api/users/my/emotion-calendar?year=2026&month=3" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sS "${BASE_URL}/api/users/my/emotion-heatmap?days=30" \
  -H "Authorization: Bearer ${TOKEN}"
```

验收要点：
- `/api/account/info` 返回 `user_id / nickname / avatar_url / is_anonymous`
- 情绪日历与热力图都返回 `days` 结构
- `days` 中的单日记录应携带真实 `score/raw_score/mood` 之一；坏载荷不能再被前端落成默认中性记录或空图
- 前端资料页、情绪日历页、热力图页能直接消费返回数据；若接口失败，页面应进入显式失败态或 stale 态，而不是展示“成功但空数据”

### 2.4 守望者与灯火计划

```bash
curl -sS ${BASE_URL}/api/guardian/stats \
  -H "Authorization: Bearer ${TOKEN}"

curl -sS ${BASE_URL}/api/guardian/insights \
  -H "Authorization: Bearer ${TOKEN}"

curl -sS ${BASE_URL}/api/vip/status \
  -H "Authorization: Bearer ${TOKEN}"
```

验收要点：
- 守望者统计页能拿到 `resonance_points / can_transfer_lamp / role`
- 灯火计划页能拿到 `is_vip / days_left`
- 当前云端统一走公网网关 `http://121.41.195.165`，不再各端直连后端容器端口

## 3. 常见故障排查

| 现象 | 先查什么 | 处理方式 |
|------|----------|----------|
| 三端连不上 | 网关地址是否统一到 `http://121.41.195.165` | 检查 `frontend/lib/utils/app_config.dart` 和 `admin/vite.config.ts` |
| 登录后资料页异常 | `/api/account/info` 返回字段是否完整 | 确认 `user_id`、`is_anonymous`、`avatar_url` 都存在 |
| 头像上传成功但页面不刷新 | 上传返回是否带 `url` 或 `relative_url` | 先看 `/api/media/upload` 响应，再看 `/api/account/avatar` 是否落库 |
| 管理端空白或列表为空 | `/api/admin/*` 是否通过网关返回有效 `data/items/list` | 打开浏览器网络面板，核对响应结构 |
| Flutter 页面不更新 | WebSocket 是否重连、对应事件是否广播 | 检查 `/ws/broadcast` 与页面监听事件名 |
| Docker 冒烟失败 | 服务容器是否都 healthy | `docker compose ps` 后再看 `gateway` 和 `backend` 日志 |

## 4. 交付前检查清单

- `./scripts/verify-2c2g.sh` 通过
- `./scripts/docker-test.sh` 通过
- 登录、投石、涟漪、纸船、好友、守望者、灯火、情绪日历、热力图至少手动走一遍
- 管理后台首页、内容列表、举报列表、系统配置页至少手动打开一次
