# 端到端测试与故障排查手册

> 本文档涵盖 HeartLake 的测试体系概览、核心链路验证用例、故障排查方法及交付前检查清单。

## 1. 测试体系概览

| 层级 | 框架 | 测试文件数 | 覆盖范围 |
|------|------|-----------|----------|
| 后端单元 / 集成测试 | Google Test | 57 | AI 引擎、加密、工具类、领域逻辑、并发安全 |
| 移动端单元测试 | Dart test | 33 | 模型序列化、端侧 AI、服务层、压力测试 |
| 管理后台测试 | Vitest | 25 | 组件渲染、API 客户端、Store、WebSocket、压力测试 |
| 端到端冒烟测试 | Shell + Docker | — | 全链路连通性验证 |

## 3. 核心链路验证用例

### 用例 A：认证 → 发布 → 互动

```bash
# 1. 匿名登录
TOKEN=$(curl -sS -X POST http://localhost:8080/api/auth/anonymous \
  -H 'Content-Type: application/json' \
  -d '{"device_id":"e2e_test"}' | jq -r '.data.token')

# 2. 发布石头
curl -sS -X POST http://localhost:8080/api/stones \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"今天心情不错","mood_type":"happy"}'

# 3. 查询湖面石头流
curl -sS "http://localhost:8080/api/lake/stones?page=1&page_size=10" \
  -H "Authorization: Bearer $TOKEN"

# 4. 点涟漪（幂等操作）
curl -sS -X POST http://localhost:8080/api/stones/{stone_id}/ripples \
  -H "Authorization: Bearer $TOKEN"
```

验收标准：全部返回 `code: 0`，涟漪重复操作返回 `already_rippled: true`。

### 用例 B：好友与消息

```bash
# 1. 查询好友列表
curl -sS http://localhost:8080/api/friends \
  -H "Authorization: Bearer $TOKEN"

# 2. 发送消息（要求亲密度 >= 12）
curl -sS -X POST http://localhost:8080/api/friends/{friend_id}/messages \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"你好"}'

# 3. 删除好友（软删除）
curl -sS -X DELETE http://localhost:8080/api/friends/{friend_id} \
  -H "Authorization: Bearer $TOKEN"
# 预期返回 mode: intimacy_auto_hidden

# 4. 删除后发送消息应返回 403
curl -sS -X POST http://localhost:8080/api/friends/{friend_id}/messages \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"test"}'
# 预期返回 code: 403
```

## 4. 故障排查

### 4.1 后端启动失败

| 症状 | 可能原因 | 排查方法 |
|------|----------|----------|
| 启动后立即退出 | PostgreSQL 不可达 | `pg_isready -h localhost -p 5432` |
| 启动后立即退出 | Redis 不可达 | `redis-cli -a HeartLake ping` |
| 启动后立即退出 | PASETO_KEY 长度不足 | 检查 `.env`，确保 >= 32 字节 |
| 端口绑定失败 | 端口被占用 | `ss -ltnp | grep :8080` |

### 4.2 接口返回 500

排查步骤：

1. 查看后端日志中的首条异常信息
2. 确认数据库迁移已完整执行
3. 检查请求体字段类型是否匹配

```bash
docker compose logs backend --tail=50
# 或
tail -50 backend/logs/heartlake.log
```

### 4.3 前端页面空白

1. 确认后端 8080 端口正在监听
2. 检查浏览器控制台是否有网络请求错误
3. 强制刷新页面：`Ctrl + Shift + R`（清除 Service Worker 缓存）
4. 重启 Flutter Web 服务

### 4.4 WebSocket 连接失败

1. 确认连接 URL 携带了有效令牌
2. 检查服务端日志是否有鉴权失败记录
3. 确认心跳机制正常运行
4. 排查是否有代理或防火墙拦截 WebSocket 升级请求

### 4.5 Docker 容器排查

```bash
docker compose ps                          # 查看容器状态
docker compose logs backend --tail=100     # 后端日志
docker compose logs postgres --tail=50     # 数据库日志
docker compose restart backend             # 重启单个容器
docker compose down && docker compose up -d --build  # 完全重建
```

### 4.6 数据库排查

```bash
psql -h localhost -U postgres -d heartlake -c "SELECT 1"   # 验证连通性
psql -h localhost -U postgres -d heartlake -c "\dt"         # 查看表结构
cd backend && ./start.sh                                     # 重新执行迁移
```
