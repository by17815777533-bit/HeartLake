### ASAN 内存安全验证

使用 AddressSanitizer 构建，检测内存越界、use-after-free、内存泄漏：

```bash
mkdir -p build-asan && cd build-asan
cmake -DCMAKE_BUILD_TYPE=Debug \
  -DCMAKE_CXX_FLAGS="-fsanitize=address -fno-omit-frame-pointer" ..
make -j$(nproc)
ctest --output-on-failure
```

目标：ASAN 构建下全部测试通过，0 内存错误。

## 端到端测试

### Docker 环境全量测试

```bash
./scripts/docker-test.sh
```

执行流程：
1. 启动全部 Docker 容器
2. 等待健康检查通过
3. 冒烟测试（健康检查 → 匿名登录 → 核心接口）
4. 容器内运行后端 `ctest`
5. 容器内运行前端 `flutter test`
6. 容器内运行管理后台 `npx vitest run`
7. 输出汇总报告

## 故障排查速查

| 症状 | 可能原因 | 排查方法 |
|------|----------|----------|
| 后端测试编译失败 | 缺少依赖库 | 检查 CMake 输出，安装缺失的 -dev 包 |
| ctest 部分失败 | 数据库未启动 | 确认 PostgreSQL/Redis 可达 |
| flutter test 失败 | 依赖未安装 | `flutter pub get` |
| vitest 失败 | node_modules 过期 | `cd admin && npm install` |
| ASAN 报错 | 内存安全问题 | 根据 ASAN 输出定位源码行号 |
| 压力测试崩溃 | 并发安全问题 | 检查 mutex/atomic 使用，用 TSAN 辅助定位 |
| docker-test 超时 | 容器启动慢 | 增加健康检查等待时间，检查资源限制 |
