# 测试验证与压测手册

本文记录当前可执行的验证入口，以及最近一次云端压测的真实结果。

## 1. 当前验证命令

```bash
./scripts/verify-2c2g.sh
./scripts/docker-test.sh
VERIFY_ONNX_SMOKE=1 ./scripts/verify-onnx-smoke.sh
```

### `verify-2c2g.sh`

- 后端 Release 构建
- 管理端 `npm run build`
- Flutter `flutter analyze`

### `docker-test.sh`

- 网关健康检查
- 管理后台首页检查
- 匿名登录
- 账号隐私读写
- 湖面流、投石、涟漪
- 情绪日历、热力图
- 好友列表
- 推荐接口
- 客户端 HTTP + WebSocket smoke

## 2. 云端压测环境

- 服务器：`heartlake-server`
- 目录：`/root/HeartLake`
- 压测发起地址：服务器本机 `127.0.0.1`
- 网关端口：`3000`
- 压测窗口：
  - 5 分钟混合 HTTP soak
  - 120 并发 WebSocket 连接压测
  - PostgreSQL 慢 SQL 观测窗口 `20ms`

- PostgreSQL 存在 `collation version mismatch`
- 当前未触发慢 SQL，但以上表需要持续盯索引和执行计划

## 7. 当前性能结论

- 当前 2C2G 单机部署已达到上线级性能
- 高频读链和常规写链稳定在几十毫秒内
- 高级算法链已实压，不是绕过算法的空跑
- WebSocket 当前连接能力稳定，120 并发建连没有错误
