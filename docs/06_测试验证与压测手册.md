# 校验与压测手册

> 2026-03 起，仓库已移除低质量三端测试目录。当前验收策略是“生产代码路径优先”，以构建、联调冒烟和可选模型烟雾验证为主。

这份文档偏“验证基线与压测建议”。如果你准备正式发版，请把 [10_上线检查清单](./10_上线检查清单.md) 作为最后执行入口。

## 1. 当前校验方式

| 校验项 | 命令 | 目标 |
|--------|------|------|
| 2C2G 构建校验 | `./scripts/verify-2c2g.sh` | 后端 Release 构建、管理端 build、Flutter analyze |
| Docker 联调冒烟 | `./scripts/docker-test.sh` | 校验网关、API、管理端和核心业务接口连通 |
| ONNX 烟雾验证 | `VERIFY_ONNX_SMOKE=1 ./scripts/verify-onnx-smoke.sh` | 校验 ONNX Runtime、模型文件、词表和默认推理链路 |

## 2. 2C2G 校验说明

`./scripts/verify-2c2g.sh` 当前执行三件事：

1. `backend` 以 `Release` 模式构建主程序
2. `admin` 执行 `npm run build`
3. `frontend` 执行 `flutter analyze`

默认资源上限：
- 后端构建并发 `2`
- 管理端 Node 内存 `512MB`
- 可选 ONNX 冒烟默认关闭

## 3. Docker 冒烟范围

`./scripts/docker-test.sh` 主要覆盖：
- 网关健康检查
- 匿名登录
- 账号资料读取与更新
- 湖面石头列表
- 投石主流程
- 涟漪互动
- 我的涟漪 / 好友列表
- 情绪日历 / 情绪热力图

这类冒烟直接覆盖当前项目最容易出问题的链路：网关代理、三端契约、媒体 URL、WebSocket 和分页集合结构。

## 4. ONNX 烟雾验证

仅在需要确认端侧情绪识别模型链路时执行：

```bash
./scripts/restore-onnx-model.sh
VERIFY_ONNX_SMOKE=1 ./scripts/verify-onnx-smoke.sh
```

适用场景：
- 新模型文件替换
- ONNX Runtime 版本升级
- 边缘 AI 默认推理链路调整

## 5. 压测建议

当前不再维护仓库内压测代码目录。若需要做压测，优先直接针对生产接口做：
- `/api/lake/stones`
- `/api/stones`
- `/api/stones/{id}/ripples`
- `/api/stones/{id}/boats`
- `/api/recommendations/trending`
- `/ws/broadcast`

压测时重点关注：
- 网关与后端连接数
- PostgreSQL 慢查询
- Redis 命中率
- WebSocket 广播延迟
- EdgeAI / ONNX 推理耗时

## 6. 交付基线

满足以下条件即可视为通过当前工程验收：
- `./scripts/verify-2c2g.sh` 通过
- `./scripts/docker-test.sh` 通过
- 如涉及 ONNX 或 AI 模型改动，`VERIFY_ONNX_SMOKE=1 ./scripts/verify-onnx-smoke.sh` 通过
- 关键页面手动打开无白屏、无接口 500、无明显契约错位
