## 4. 后端实现

### 4.1 启动入口

- 主入口：`backend/src/main.cpp`
- 架构装配：`backend/include/infrastructure/ArchitectureBootstrap.h`
- 应用服务工厂：`backend/src/application/ApplicationServiceFactory.cpp`

### 4.2 启动顺序

`main.cpp` 启动链如下：

1. 加载 `.env` 和配置文件路径。
2. 校验 `PASETO_KEY`、`DB_PASSWORD` 等关键环境变量。
3. 计算线程数并应用低配机默认参数。
4. 配置监听地址、日志、静态目录和上传目录。
5. 注册异常处理、CORS、安全响应头、认证和链路追踪。
6. 调 `ArchitectureBootstrap::initialize()`，装配缓存、事件总线、仓储和领域服务。
7. 初始化 `RBACManager`、内容过滤、AI、Embedding、EdgeAI、共鸣、推荐、RAG、限流、Redis、FriendshipTTL、ResonanceSearch。
8. 按环境变量决定是否启动 `LakeGodGuardianService`、`EmotionTrackingService`、`UserFollowUpService`、WebSocket 心跳。
9. `app.run()` 开始接流量。

### 4.3 分层职责

- `interfaces/`：控制器和 WebSocket 入口，只做鉴权、参数校验、响应封装。
- `application/`：业务编排、跨仓储写链、事件分发。
- `domain/`：实体、仓储接口、领域规则和领域服务。
- `infrastructure/`：数据库、缓存、AI、向量检索、后台任务、过滤器、媒体存储。
- `utils/`：PASETO、RBAC、统一响应、实时事件、环境工具、配置存储。

### 4.7 八大 Edge AI 子系统

`backend/include/infrastructure/ai/EdgeAIEngine.h` 把 Edge AI 定义成八大核心子系统的统一门面：

1. `SentimentAnalyzer`
   负责轻量级情感分析，采用规则、词典、统计结果三层融合，支持缓存和 ONNX 优先推理。
2. `ContentModerator`
   负责本地文本审核，组合 AC 自动机敏感词检测和心理风险评估。
3. `EmotionPulseDetector`
   负责实时情绪脉搏，维护滑动窗口、EWMA 快照和历史脉搏序列。
4. `FederatedLearner`
   负责联邦学习聚合，支持本地模型提交、FedAvg、梯度裁剪和 DP 噪声协同。
5. `EdgeDifferentialPrivacy`
   负责差分隐私预算和噪声注入，支持 Laplace 与 Gaussian 两种机制。
6. `HNSWIndex`
   负责本地向量近邻检索和候选召回，是向量搜索和语义缓存的重要基础件。
7. `ModelQuantizer`
   负责 INT8 对称量化和量化矩阵推理，用于低资源下的本地模型执行。
8. `EdgeNodeMonitor`
   负责边缘节点健康监控，提供熔断和自适应负载信息。

### 4.8 AI 上层能力

八大 Edge AI 子系统之外，还有这些上层 AI 能力：

- `AIService`
  统一调度情感分析、内容审核、湖神回复、纸船回复、石头评论和嵌入生成。
- `AdvancedEmbeddingEngine`
  负责文本嵌入生成，供推荐、共鸣、湖神上下文和向量搜索复用。
- `RecommendationEngine`
  负责个性化推荐，融合 User-based CF、Item-based CF、Content-based 推荐和探索策略。
- `EmotionResonanceEngine`
  负责四维共鸣匹配，核心公式是语义相似、情绪轨迹、时间衰减和多样性奖励的加权。
- `DualMemoryRAG`
  负责湖神双记忆系统，组合短期对话记忆、长期情绪画像、RAG 提示词和超时降级。
- `ResonanceSearchService`
  负责把嵌入检索、共鸣分数和自然语言原因拼成最终共鸣结果。
- `LakeGodGuardianService`
  负责零互动石头巡检、自动纸船关怀和湖神守护触发。
- `EmotionTrackingService`
  负责情绪轨迹采样和趋势写入。
- `UserFollowUpService`
  负责回访与跟进。

## 9. 关键链路

- 内容链：投石 -> 入库 -> 广播 -> 列表刷新。
- 互动链：涟漪 / 纸船 -> 入库 -> 统计回写 -> 通知 / 广播。
- 关系链：自动关系计算 -> 好友 / 临时好友列表 -> 私信。
- AI 链：情绪分析 -> 推荐 / 共鸣 / 湖神 -> 显式结果或显式降级。
- 管理链：后台查询 -> 统计聚合 -> 实时推送 -> 页面 stale/warning 状态。

### AI 功能细分

- 情感分析：`EdgeAIController::analyzeLocal`、`EdgeAIService.analyzeSentiment`、`EdgeAIProvider.classifyText`
- 内容审核：`EdgeAIController::moderateLocal` 和发布前审核链
- 情绪脉搏：`EdgeAIController::getEmotionPulse`、`EmotionPulseWidget`、`EdgeAI.vue`
- 差分隐私：`EdgeAIController::getPrivacyBudget` 和管理端预算视图
- 联邦学习：`EdgeAIController::federatedAggregate` 和 `EdgeAI.vue` 聚合操作
- 推荐：`RecommendationController`、`VectorSearchController`、`discover_screen`、`personalized_screen`
- 共鸣：`EmotionResonanceEngine`、`ResonanceSearchService`、石头详情和高级推荐链
- 湖神与洞察：`GuardianController::getEmotionInsights`、`GuardianController::chat`、`DualMemoryRAG`

## 10. 工程约束

- 失败不伪装成空数据。
- 集合接口统一输出标准壳。
- 实时事件统一输出标准事件壳。
- 写链成功后才修改本地状态。
- AI、推荐和共鸣失败必须可见。
