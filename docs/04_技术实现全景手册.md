## 3. 后端实现

### 3.1 启动入口

- 主入口：`backend/src/main.cpp`
- 架构装配：`backend/include/infrastructure/ArchitectureBootstrap.h`
- 应用服务工厂：`backend/src/application/ApplicationServiceFactory.cpp`

### 3.2 启动顺序

`main.cpp` 当前启动链如下：

1. 加载 `.env` 和配置文件路径。
2. 校验 `PASETO_KEY`、`DB_PASSWORD` 等关键环境变量。
3. 计算线程数并应用低配机默认参数。
4. 配置监听地址、日志、静态目录和上传目录。
5. 注册异常处理、CORS、安全响应头、认证和链路追踪。
6. 调 `ArchitectureBootstrap::initialize()`，装配缓存、事件总线、仓储和领域服务。
7. 初始化 `RBACManager`、内容过滤、AI、Embedding、EdgeAI、共鸣、推荐、RAG、限流、Redis、FriendshipTTL、ResonanceSearch。
8. 按环境变量决定是否启动 `LakeGodGuardianService`、`EmotionTrackingService`、`UserFollowUpService`、WebSocket 心跳。
9. `app.run()` 开始接流量。

### 3.3 分层职责

- `interfaces/`：控制器和 WebSocket 入口，只做鉴权、参数校验、响应封装。
- `application/`：业务编排、跨仓储写链、事件分发。
- `domain/`：实体、仓储接口、领域规则和领域服务。
- `infrastructure/`：数据库、缓存、AI、向量检索、后台任务、过滤器、媒体存储。
- `utils/`：PASETO、RBAC、统一响应、实时事件、环境工具、配置存储。

## 4. 移动端实现

### 4.1 启动入口

- 应用入口：`frontend/lib/main.dart`
- 路由入口：`frontend/lib/router/app_router.dart`
- 配置入口：`frontend/lib/utils/app_config.dart`
- 依赖注入：`frontend/lib/di/service_locator.dart`

### 4.2 启动顺序

`main.dart` 当前顺序：

1. `runZonedGuarded` 包裹全局异常。
2. 注册 `FlutterError.onError`、`platformDispatcher.onError`、`ErrorWidget.builder`。
3. 执行 `setupServiceLocator()`。
4. 执行 `appConfig.initialize()`。
5. 启动 `CacheService().startAutoCleanup()`。
6. 配置 `ApiClient` 的 401 和 token 刷新回调。
7. 用 `MultiProvider` 挂载全局状态。

### 5.3 状态、HTTP 和实时链

## 8. 当前关键链路

- 内容链：投石 -> 入库 -> 广播 -> 列表刷新。
- 互动链：涟漪 / 纸船 -> 入库 -> 统计回写 -> 通知 / 广播。
- 关系链：自动关系计算 -> 好友 / 临时好友列表 -> 私信。
- AI 链：情绪分析 -> 推荐 / 共鸣 / 湖神 -> 显式结果或显式降级。
- 管理链：后台查询 -> 统计聚合 -> 实时推送 -> 页面 stale/warning 状态。

## 9. 当前工程约束

- 失败不伪装成空数据。
- 集合接口统一输出标准壳。
- 实时事件统一输出标准事件壳。
- 写链成功后才修改本地状态。
- AI、推荐和共鸣失败必须可见。
