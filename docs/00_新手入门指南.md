# 新手入门指南

HeartLake 当前是一套三端联动的匿名情绪社区系统。你只需要先记住一组线上事实：

- 公网入口：`http://121.41.195.165`
- API：`http://121.41.195.165/api`
- 管理后台：`http://121.41.195.165/admin/`
- WebSocket：`ws://121.41.195.165/ws/broadcast`
- 云端目录：`/root/HeartLake`

## 项目组成

- 移动端：Flutter
- 管理端：Vue 3 + Vite
- 后端：Drogon + C++20
- 数据层：PostgreSQL + Redis + 向量检索
- AI 链：推荐、共鸣搜索、EdgeAI、湖神
- 实时链：WebSocket 广播和房间消息

## 仓库布局

- `frontend/`：移动端
- `admin/`：管理端
- `backend/`：后端
- `docs/`：手册
- `scripts/`：部署、验证、联调
- `datasets/`：离线参考数据

## 当前工作原则

- 以当前云端和当前代码为准。
- 失败必须显式暴露。
- 三端共享同一套接口契约。
- 未开放能力不写进功能说明。
