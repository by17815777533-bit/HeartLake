# 新手入门指南

> 本文档面向首次接触 HeartLake 项目的开发者，提供从获取源码到完整运行全部服务的逐步操作指引。

## 前置条件

在开始之前，请确认本机已安装以下软件：

| 软件 | 最低版本 | 验证命令 | 用途 |
|------|----------|----------|------|
| Git | 2.30+ | `git --version` | 克隆仓库 |
| Docker Engine | 20.10+ | `docker --version` | 容器运行时 |
| Docker Compose | v2+ | `docker compose version` | 多容器编排 |

硬件建议：

- CPU：2 核以上
- 内存：`all` 模式至少 4GB；`lite` / `server-lite` 模式 2GB 也可运行
- 磁盘：至少 10GB 可用空间（Docker 镜像构建 + 数据库存储）

> 如果仅需运行 `lite` / `server-lite`，2GB 内存也可运行，但更建议预留 swap。

## 第一步：获取源码

```bash
git clone https://github.com/by17815777533-bit/HeartLake.git
cd HeartLake
```

确认目录结构正确：

```bash
ls
# 应看到：backend/  frontend/  admin/  docker-compose.yml  scripts/  docs/  ...
```

## 第二步：配置环境变量

```bash
cp .env.example .env
```

这是模板文件，不建议直接照搬到生产环境。至少需要填写密码、密钥和管理员密码哈希。主要关注以下变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DB_PASSWORD` | 必填 | PostgreSQL 数据库密码 |
| `REDIS_PASSWORD` | 必填 | Redis 认证密码 |
| `PASETO_KEY` | 必填 | 用户端 PASETO 密钥（至少 32 字节） |
| `ADMIN_PASETO_KEY` | 必填 | 管理端 PASETO 密钥（至少 32 字节） |
| `ADMIN_PASSWORD_HASH` | 必填 | 管理员密码 PBKDF2 哈希 |
| `BACKEND_PORT` | `8080` | 后端服务端口 |
| `ADMIN_PORT` | `5173` | 管理后台端口 |
| `GATEWAY_PORT` | `3000` | Nginx 网关端口 |

完整环境变量说明参见 [README.md](../README.md#环境变量)。

## 第三步：启动服务

### 方式 A：全量启动（4GB+ 机器）

启动全部 6 个容器（PostgreSQL、Redis、Ollama、Backend、Admin、Gateway）：

```bash
./scripts/docker-up.sh all
```

首次启动需要构建 Docker 镜像，耗时约 5-15 分钟（取决于网络和机器性能）。后续启动通常在 30-60 秒内完成。

### 方式 B：精简启动（低配机器）

不启动 Ollama（节省内存），AI 对话功能降级，但 EdgeAI 本地情感分析正常运行：

```bash
./scripts/docker-up.sh lite
```

### 方式 C：仅数据库

只启动 PostgreSQL 和 Redis，适用于本地开发后端时使用：

```bash
./scripts/docker-up.sh db
```

## 第四步：确认服务状态

等待启动完成后（首次约 1-2 分钟），逐项验证：

### 4.1 检查容器运行状态

```bash
docker compose ps
```

预期输出：所有容器状态为 `Up` 或 `Up (healthy)`。

### 4.2 后端健康检查

```bash
curl -sS http://localhost:8080/api/health
```

预期响应：

```json
{"code":0,"message":"success","data":{"status":"ok"}}
```

### 4.3 管理后台访问

浏览器打开 `http://localhost:5173`，应看到登录页面。

管理员账号来自 `.env` 中的 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD_HASH`，不再假定固定默认密码。

### 4.4 统一网关

浏览器打开 `http://localhost:3000`，应看到网关状态页。

### 4.5 运行自动化验证脚本

```bash
./scripts/docker-test.sh
```

该脚本会自动执行健康检查、匿名登录、核心接口调用等冒烟测试，全部通过即表示服务运行正常。

## 第五步：导入演示数据（可选）

首次启动后数据库为空（仅有表结构）。如需体验完整功能，可导入演示数据：

```bash
# 重置数据库并导入种子数据（包含示例石头、用户、互动记录）
./scripts/reset_all_data_and_seed.sh
```

创建一个具有完整权限的演示账号（VIP + Guardian 权限）：

```bash
./scripts/create_full_access_account.sh
```

> 注意：`reset_all_data_and_seed.sh` 会清空现有数据。如仅需创建表结构而不导入数据，使用 `--schema-only` 参数。

## 第六步：体验核心功能

### 通过 curl 体验 API

```bash
# 1. 匿名登录，获取令牌
TOKEN=$(curl -sS -X POST http://localhost:8080/api/auth/anonymous \
  -H 'Content-Type: application/json' \
  -d '{"device_id":"my_first_test"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")

echo "Token: $TOKEN"

# 2. 发布一颗石头
curl -sS -X POST http://localhost:8080/api/stones \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"content":"第一次使用心湖，感觉很新奇","mood":"happy","tags":["新手"]}'

# 3. 查看湖面石头流
curl -sS "http://localhost:8080/api/lake/stones?page=1&page_size=5"

# 4. AI 情感分析
curl -sS -X POST http://localhost:8080/api/edge-ai/analyze \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"text":"今天天气真好，心情也跟着好起来了"}'

# 5. 查看情绪脉搏
curl -sS http://localhost:8080/api/edge-ai/emotion-pulse \
  -H "Authorization: Bearer $TOKEN"
```

### 通过管理后台体验

1. 浏览器打开 `http://localhost:5173`
2. 使用你在 `.env` 中配置的管理员账号登录
3. Dashboard 页面可查看用户增长、情绪分布、热门话题等统计数据
4. 左侧菜单可进入用户管理、内容审核、敏感词管理等功能模块

## 停止服务

```bash
# 停止所有容器（保留数据卷）
./scripts/docker-down.sh

# 停止并清除所有数据卷（数据库数据将丢失）
./scripts/docker-down.sh --purge
```

## 常见问题

### Q: docker-up.sh 报权限错误

```bash
chmod +x scripts/*.sh
```

### Q: 后端容器反复重启

查看后端日志定位原因：

```bash
docker compose logs backend --tail=50
```

常见原因：
- PostgreSQL 尚未就绪：等待 30 秒后重试，健康检查机制会自动处理
- 端口被占用：检查 `ss -ltnp | grep -E ':8080|:5432|:6379'`，关闭占用端口的进程

### Q: 首次构建镜像非常慢

后端 Docker 镜像需要编译 C++ 项目，首次构建耗时较长属于正常现象。构建完成后镜像会被缓存，后续启动无需重新编译。

如果网络不稳定导致依赖下载失败，可重试：

```bash
docker compose build --no-cache backend
```

### Q: Ollama 模型未加载

Ollama 容器启动后需要手动拉取模型：

```bash
docker exec -it heartlake-ollama ollama pull qwen2.5:1.5b
```

未加载模型时，AI 对话功能返回降级回复，但不影响 EdgeAI 本地情感分析等功能。

### Q: 管理后台页面空白

1. 确认后端服务正常运行：`curl -sS http://localhost:8080/api/health`
2. 清除浏览器缓存：`Ctrl + Shift + R` 强制刷新
3. 检查浏览器控制台是否有网络请求错误

### Q: 端口冲突

如果默认端口被占用，在 `.env` 文件中修改对应端口：

```bash
# 例如将后端端口改为 9080
BACKEND_PORT=9080
```

修改后重启服务：

```bash
./scripts/docker-down.sh
./scripts/docker-up.sh all
```

### Q: 内存不足导致容器被 OOM Kill

减少启动的服务数量：

```bash
# 不启动 Ollama，节省约 1-2GB 内存
./scripts/docker-up.sh lite
```

或在 `docker-compose.yml` 中为各服务添加内存限制。
